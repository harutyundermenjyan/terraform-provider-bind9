# BIND9 Provider

The BIND9 provider allows you to manage DNS zones and records on BIND9 servers through the [BIND9 REST API](https://gitlab.com/Dermenjyan/bind9-api). This provider supports comprehensive DNS management including zone creation, record management for all DNS record types, and DNSSEC key management.

## Features

- **Zone Management** - Create, update, and delete DNS zones (master, slave, forward, stub)
- **Record Management** - Full CRUD operations for 20+ DNS record types
- **DNSSEC Support** - Generate keys, sign zones, export DS records
- **Data Sources** - Query existing zones and records
- **Import Support** - Import existing resources into Terraform state

## Example Usage

```terraform
terraform {
  required_providers {
    bind9 = {
      source  = "harutyundermenjyan/bind9"
      version = "~> 1.0"
    }
  }
}

# Configure the provider
provider "bind9" {
  endpoint = "http://dns.example.com:8080"
  api_key  = var.bind9_api_key
}

# Create a zone
resource "bind9_zone" "example" {
  name = "example.com"
  type = "master"

  soa_mname   = "ns1.example.com"
  soa_rname   = "hostmaster.example.com"
  default_ttl = 3600

  nameservers = ["ns1.example.com", "ns2.example.com"]
  ns_addresses = {
    "ns1.example.com" = "10.0.1.10"
    "ns2.example.com" = "10.0.1.11"
  }

  allow_update = ["key ddns-key"]
}

# Create an A record
resource "bind9_record" "www" {
  zone    = bind9_zone.example.name
  name    = "www"
  type    = "A"
  ttl     = 300
  records = ["10.0.2.100"]
}
```

## Authentication

The provider supports two authentication methods:

### API Key Authentication (Recommended)

```terraform
provider "bind9" {
  endpoint = "http://dns.example.com:8080"
  api_key  = var.bind9_api_key
}
```

### Username/Password Authentication (JWT)

```terraform
provider "bind9" {
  endpoint = "http://dns.example.com:8080"
  username = var.bind9_username
  password = var.bind9_password
}
```

## Environment Variables

You can configure the provider using environment variables:

| Variable | Description |
|----------|-------------|
| `BIND9_ENDPOINT` | API endpoint URL |
| `BIND9_API_KEY` | API key for authentication |
| `BIND9_USERNAME` | Username for JWT authentication |
| `BIND9_PASSWORD` | Password for JWT authentication |

```shell
export BIND9_ENDPOINT="http://dns.example.com:8080"
export BIND9_API_KEY="your-api-key-here"
```
